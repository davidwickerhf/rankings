# Reproducing The Repository Outputs

This repository exposes two practical levels of reproduction:

1. regenerate the paper-facing outputs from the fixed centrality bundles already
   retained in the repository;
2. regenerate earlier-stage metadata and network artifacts from the canonical
   metadata source.

The top-level `Makefile` provides the shortest entry points.

## Environment

Use the project virtual environment:

```bash
source venv/bin/activate
```

## Fast Validation

Validate that the active Python scripts import and compile:

```bash
make smoke
```

## Regenerate The Paper Outputs

This reruns the supporting table, the correlation matrix, and the paper-facing
analysis scripts from the fixed `total_df.csv` bundles already stored under
`results/fixed-merged-subarticles-edges/`.

```bash
make paper-results
```

This target runs:

- `scripts/generate/01_generate_network_statistics.py`
- `scripts/generate/03_generate_correlation_matrix.py`
- all active scripts under `scripts/analysis/`

## Regenerate Load And Network Artifacts

This reruns the metadata-processing and network-construction stages from the
canonical metadata source file:

```bash
make from-metadata
```

This target runs:

- `scripts/load/process_metadata.py`
- `scripts/load/extract_edges.py`
- `scripts/generate/00_generate_merged_article_edge_networks.py`

## Rebuild Centrality Bundles

The full centrality-build step is available as a script, but it is intentionally
not wrapped into the default `make paper-results` target because it can be
computationally heavy and may be rerun selectively by network family.

Representative example:

```bash
venv/bin/python scripts/generate/02_build_centrality_results.py \
  --network-dir networks/merged-article-edges/split-unbalanced \
  --output-dir results/rebuilt/split-unbalanced \
  --min-nodes 50
```

Use this script when you want to reconstruct `total_df.csv`,
`analysis_results.json`, and the per-network plots for a particular network
collection.

## Where To Look After Running

- `results/load/`
  - Load-stage derived artifacts.
- `networks/merged-article-edges/`
  - Active network family.
- `results/analysis/`
  - Paper-facing outputs regenerated by `make paper-results`.
- `results/fixed-merged-subarticles-edges/`
  - Fixed centrality bundles consumed by the analysis scripts.

For a paper-to-output mapping, see `docs/TRACEABILITY.md`.
